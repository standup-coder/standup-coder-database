# 技术辅导方法论工具包

## 1. 辅导概述

### 1.1 什么是技术辅导
技术辅导(Technical Coaching)是指通过系统化的方法帮助技术人员提升专业能力、解决职业困惑、实现职业目标的过程。不同于简单的知识传授，技术辅导更注重：
- **启发思考**：引导被辅导者自主发现问题和解决方案
- **能力构建**：帮助建立可持续成长的能力体系
- **职业发展**：结合个人特质制定长期发展路径

### 1.2 辅导的适用场景
- **新人 onboarding**：帮助新员工快速融入团队和技术栈
- **技能提升**：针对特定技术领域的能力补强
- **职业转型**：从前端转全栈、从开发转架构等
- **领导力培养**：从 IC 向 Tech Lead 或 Manager 过渡
- **绩效改进**：帮助低绩效员工找到问题并改进

### 1.3 辅导者与导师的区别

| 维度 | 导师(Mentor) | 辅导者(Coach) |
|------|-------------|--------------|
| **核心关系** | 经验分享者 | 提问引导者 |
| **沟通方式** | "我建议你..." | "你认为...?" |
| **目标设定** | 导师提供方向 | 被辅导者自主发现 |
| **时间周期** | 长期关系 | 短期聚焦 |
| **关注焦点** | 职业发展全局 | 具体问题解决 |

## 2. 辅导方法论模型

### 2.1 GROW 模型
GROW 是最经典的技术辅导框架：

```
Goal（目标）→ Reality（现状）→ Options（选项）→ Will（意愿）
```

**实践步骤**：
1. **Goal**：明确本次辅导要达成的具体目标
   - "我们今天讨论，你希望收获什么？"
   - "三个月内，你想在这个领域达到什么水平？"

2. **Reality**：客观分析当前状况
   - "目前你在XX技术上的实际情况是？"
   - "是什么阻碍了你达成目标？"

3. **Options**：探索所有可能的解决方案
   - "如果资源不限，你会怎么做？"
   - "你听说过哪些方法？它们的优缺点是什么？"

4. **Will**：确定行动计划和承诺
   - "你打算第一步做什么？"
   - "什么时候完成？需要什么支持？"

### 2.2 OSCAR 模型
适用于绩效改进和能力提升场景：

```
Outcome（结果）→ Situation（情境）→ Choices（选择）→ Actions（行动）→ Review（回顾）
```

### 2.3 3A 辅导模型
适用于技术能力专项提升：

- **Awareness（认知）**：帮助对方意识到能力差距
- **Action（行动）**：制定具体的学习和实践计划
- **Accountability（问责）**：定期回顾进展并提供反馈

## 3. 不同场景的辅导策略

### 3.1 新人入职辅导
**阶段划分**：

| 阶段 | 时间 | 重点内容 | 辅导频率 |
|------|------|---------|---------|
| **适应期** | 第1周 | 环境熟悉、工具配置、代码规范 | 每天 |
| **学习期** | 第2-4周 | 业务理解、技术栈掌握、首任务完成 | 每周2-3次 |
| **独立期** | 第2-3月 | 独立完成任务、参与技术讨论 | 每周1次 |
| **成长期** | 第3-6月 | 承担复杂任务、开始指导他人 | 双周1次 |

**关键检查点**：
- [ ] 开发环境能否独立搭建
- [ ] 能否独立完成一个 Bug 修复并上线
- [ ] 能否独立完成一个功能需求
- [ ] 是否理解团队的代码规范和评审标准
- [ ] 是否能主动提出技术改进建议

### 3.2 技术专项提升辅导
**典型场景**：帮助中级工程师掌握分布式系统设计

**辅导计划模板**：
```
【辅导目标】掌握微服务架构设计能力
【时间周期】3个月
【里程碑】
  - Month 1：理论学习 + 小系统改造
  - Month 2：中型项目架构设计
  - Month 3：评审和优化
【学习资源】
  - 《Designing Data-Intensive Applications》
  - 内部架构文档和案例
  - 业界公开的技术博客和演讲
【实践任务】
  - 重构一个单体服务为微服务
  - 设计评审会议
【评估标准】
  - 能独立完成服务拆分方案
  - 能通过团队架构评审
```

### 3.3 职业转型辅导
**常见转型路径**：
- 前端工程师 → 全栈工程师
- 后端工程师 → 架构师
- 技术 IC → 技术管理者
- 传统开发 → AI/数据工程

**辅导重点**：
1. **能力差距分析**：明确当前能力 vs 目标岗位要求
2. **学习路径规划**：制定3-6个月的专项提升计划
3. **实践机会创造**：在现有工作中寻找练手机会
4. **心理建设**：应对转型期的不确定性和挫败感

## 5. 常见问题与应对策略

### 5.1 被辅导者缺乏动力
**表现**：敷衍应付、不完成任务、消极态度

**应对策略**：
- 重新确认目标是否是对方真正想要的
- 探索背后的障碍（恐惧、不自信、外部压力）
- 将大目标拆解为更小、更容易达成的里程碑
- 适当引入竞争或展示机制

### 5.2 技术能力差距过大
**表现**：辅导内容对方难以理解，进度缓慢

**应对策略**：
- 降低起点，从更基础的概念开始
- 使用类比和可视化方式解释复杂概念
- 推荐针对性的学习资源
- 考虑引入更有针对性的专项培训

### 5.3 辅导效果不明显
**表现**：多次辅导后能力未见明显提升

**应对策略**：
- 评估目标是否合理（SMART原则）
- 检查是否有足够的实践机会
- 考虑换辅导方式（如结对编程代替讲解）
- 评估是否存在非技术因素（如态度、工作环境）

### 5.4 被辅导者抵触反馈
**表现**： defensive，不接受批评建议

**应对策略**：
- 先建立信任关系，再提改进建议
- 使用SBI模型（Situation-Behavior-Impact）描述问题
- 多问少讲，让对方自己发现问题
- 强调成长型思维（Growth Mindset）

## 6. 辅导效果评估

### 6.1 量化指标
- **代码质量**：Bug率、Code Review通过率、测试覆盖率
- **交付效率**：任务完成时间、估时准确度
- **技术影响力**：技术分享次数、文档贡献量、MR/PR数量
- **团队反馈**：360度评估中同事评价

### 6.2 质性指标
- 解决问题的思路是否更加系统化
- 是否主动承担更有挑战性的任务
- 是否开始帮助和指导其他同事
- 对技术的热情和好奇心是否提升

### 6.3 评估周期
- **短期**（1-3个月）：行为和知识层面的变化
- **中期**（3-6个月）：能力和技能层面的提升
- **长期**（6-12个月）：绩效和职业发展的结果
