# 创建深圳科技企业清单

## 1. 项目结构规划
- 创建 `list` 文件夹用于存放所有markdown清单文件
- 根目录保留 `README.md`，并更新其内容

## 2. 主题文件创建
按照科技行业的不同维度，创建以下主题文件：

| 文件名 | 主题 | 覆盖范围 |
|--------|------|----------|
| `ai.md` | 人工智能 | 深圳AI企业，包括机器学习、深度学习、计算机视觉、自然语言处理等 |
| `internet.md` | 互联网 | 深圳互联网企业，包括电商、社交、内容、生活服务等 |
| `software.md` | 软件 | 深圳软件企业，包括企业软件、SaaS、应用软件等 |
| `cloud.md` | 云计算 | 深圳云计算企业，包括IaaS、PaaS、云服务提供商等 |
| `3c.md` | 3C电子 | 深圳3C电子企业，包括消费电子、硬件设备、智能终端等 |
| `other-tech.md` | 其他科技领域 | 深圳其他科技企业，包括物联网、区块链、大数据、智能制造等 |

## 3. 内容结构设计
每个主题文件采用统一的内容结构：
- 主题介绍
- 企业列表（包含企业名称、简称、成立时间、业务领域、代表产品/服务、公司规模等信息）
- 参考资料

## 4. README.md 更新
- 更新README.md，添加项目说明
- 说明项目用途（求职、职业发展、行业研究等）
- 列出所有主题文件及其简要说明
- 添加贡献指南

## 5. 实施步骤
1. 创建 `list` 文件夹
2. 创建各个主题的markdown文件
3. 为每个文件填充深圳科技企业信息
4. 更新 `README.md` 文件
5. 确保所有文件格式正确，内容完整

## 6. 企业信息收集
- 确保覆盖深圳主要科技企业
- 信息来源包括企业官网、行业报告、招聘平台等
- 确保信息准确性和时效性

## 7. 格式规范
- 使用Markdown格式
- 统一的标题层级
- 清晰的表格结构
- 适当的分类和标签
