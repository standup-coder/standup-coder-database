# 快应用发布指南

## 快应用简介

快应用（Quick App）是国内安卓厂商联合推出的轻量级应用形态，无需安装，即点即用。

## 生态成员

## 开发框架

### 方案一：原生快应用

```bash
# 使用快应用官方框架：
- hap-toolkit（命令行工具）
- IDE 支持（https://www.quickapp.cn/）
- 专用语法（类 Vue/类小程序）

# 优点：性能最好，原生渲染
# 缺点：学习成本高，生态较小
```

#### 原生快应用项目结构

```bash
# 项目结构
├── src/
│   ├── App.ux           # 入口组件
│   ├── pages/
│   │   └── index/
│   │       └── index.ux    # 页面
│   └── manifest.json    # 应用配置
├── sign/               # 签名文件
├── dist/               # 编译输出
├── platform/           # 平台适配
└── manifest.json       # 项目配置
```

### 方案二：uni-app（推荐）

```bash
# 使用 uni-app 开发快应用：
- Vue 语法（上手容易）
- 一套代码多端（iOS/Android/小程序/快应用）
- 转快应用

# 官方文档：https://uniapp.dcloud.net.cn/
```

#### uni-app 开发快应用详细步骤

```bash
# 第一步：安装 HBuilderX
# 1. 下载地址：https://www.dcloud.io/hbuilderx.html
# 2. 安装到本地

# 第二步：创建 uni-app 项目
# 1. 打开 HBuilderX
# 2. 文件 → 新建 → 项目
# 3. 选择 "uni-app"
# 4. 填写项目名称
# 5. 选择模板（默认模板）
# 6. 点击创建

# 第三步：配置快应用支持
# 1. 打开 manifest.json
# 2. 点击 "快应用" 配置
# 3. 填写快应用名称和包名
# 4. 保存

# 第四步：开发页面
# 1. 在 pages/ 目录下创建页面
# 2. 使用 Vue 语法开发
# 3. 使用平台适配 API

# 第五步：编译快应用
# 1. 点击 "运行" → "运行到快应用"
# 2. 选择目标平台（华为/小米等）
# 3. 等待编译完成

# 第六步：生成 rpk 包
# 1. 点击 "发行" → "快应用"
# 2. 选择目标平台
# 3. 填写版本信息
# 4. 点击 "发行"
# 5. 在 dist/build/ 目录下生成 rpk 文件
```

### 方案三：Taro

```bash
# 使用 Taro 开发快应用：
- React 语法
- 一套代码多端（React Native/小程序/快应用）
- 转快应用

# 官方文档：https://taro.jd.com/
```

#### Taro 开发快应用详细步骤

```bash
# 第一步：安装 Node.js（需要 v12+）
# https://nodejs.org/

# 第二步：创建 Taro 项目
npx @tarojs/cli init myQuickApp
cd myQuickApp

# 第三步：安装依赖
npm install

# 第四步：配置快应用
# 修改 config/index.js
config = {
  mini: {},
  h5: {},
  quickapp: {}
}

# 第五步：开发代码
# 使用 React 语法开发
# 在 src/pages/ 下创建页面

# 第六步：编译快应用
npm run build:quickapp

# 第七步：获取 rpk 文件
# 在 dist/quickapp/ 目录下生成 rpk
```

## 发布流程

### 1. 注册开发者账号

```bash
# 各平台分别注册：
- 华为开发者平台：https://developer.huawei.com/consumer/cn/
- 小米开发者平台：https://dev.mi.com/
- OPPO 开放平台：https://open.oppommel.com/
- vivo 开发者平台：https://dev.vivo.com.cn/

# 注意：
# - 快应用可能需要单独的开发者认证
# - 部分平台支持与 App 开发者账号通用
```

### 2. 开发快应用

#### 使用 uni-app 开发（推荐）

```bash
# 第一步：创建项目
# 1. HBuilderX → 文件 → 新建 → 项目
# 2. 选择 uni-app
# 3. 填写项目名称

# 第二步：开发页面
# 在 pages/ 目录下创建 .vue 文件
# 使用 Vue 语法编写页面

# 第三步：调试
# 1. 连接手机（开启开发者模式）
# 2. 点击 "运行" → "运行到手机或模拟器"
# 3. 选择对应平台

# 第四步：适配不同平台
# 使用条件编译
<!-- #ifdef quickapp-huawei -->
<!-- 华为快应用专属代码 -->
<!-- #endif -->

<!-- #ifdef quickapp-xiaomi -->
<!-- 小米快应用专属代码 -->
<!-- #endif -->
```

### 3. 编译打包

#### uni-app 编译快应用

```bash
# 第一步：配置应用信息
# 打开 manifest.json
# 选择 "快应用" 标签
# 填写：
# - 应用名称
# - 应用包名（com.example.app）
# - 版本号
# - 图标
# - 启动页

# 第二步：设置平台差异
# 如果各平台有差异，使用条件编译

# 第三步：发行
# 1. 点击 "发行"
# 2. 选择 "快应用"
# 3. 选择目标平台（可多选）
# 4. 填写版本信息
# 5. 点击 "发行"

# 第四步：获取 rpk
# 编译完成后在项目根目录的
# dist/build/quickapp-xxx/ 目录下
# 找到 .rpk 文件
```

### 4. 各平台提交

#### 华为快应用发布

```bash
# 第一步：登录 AppGallery Connect
# 访问：https://developer.huawei.com/consumer/cn/

# 第二步：创建快应用
# 1. 选择 "元服务" → "快应用"
# 2. 点击 "创建快应用"
# 3. 填写基本信息：
#    - 快应用名称
#    - 官方网站
#    - 分类
#    - 标签

# 第三步：上传 rpk 包
# 1. 点击 "上传安装包"
# 2. 选择 .rpk 文件
# 3. 填写版本信息

# 第四步：配置隐私政策
# 1. 填写隐私政策 URL
# 2. 确保 HTTPS 可访问

# 第五步：提交审核
# 1. 点击 "提交审核"
# 2. 等待审核（1-3 天）
```

### 5. 审核与发布

```bash
# 审核周期：1-3 天

# 审核内容：
# - 功能完整性（App 必须功能可用）
# - 内容合规性（无违规内容）
# - 隐私政策（必须提供且可访问）
# - 权限使用（合理申请权限）
# - UI 规范（符合平台设计规范）

# 发布后：
# - 自动分发到各渠道
# - 更新版本需重新审核
```

## 与小程序的对比

| 特性 | 快应用 | 微信小程序 |
|------|--------|-----------|
| 安装 | 无需安装 | 无需安装 |
| 包体 | 更小（≤4MB） | ≤12MB |
| 分发 | 手机厂商商店 | 微信生态 |
| 用户触达 | 系统级入口 | 微信内入口 |
| 开发成本 | 较高 | 中等 |
| 生态封闭性 | 各大厂商独立 | 微信独占 |

## 快应用优势

```bash
# 1. 系统级入口
- 负一屏
- 应用商店
- 全局搜索

# 2. 无需安装
- 即点即用
- 不占存储

# 3. 厂商支持
- 预装在系统
- 性能优化

# 4. 推送服务
- 厂商通道
- 低功耗
```

## 快应用挑战

```bash
# 1. 碎片化
- 各厂商 API 不同
- 需要适配

# 2. 开发资源
- 需要专门投入
- 学习成本

# 3. 分发渠道
- 分散在各家
- 统一运营难
```

## 常见问题

**Q: 快应用和小程序有什么区别？**
A: 快应用是手机厂商联盟推出，系统级入口；小程序是腾讯系生态，微信内运行。

**Q: 快应用需要软著吗？**
A: 大多数渠道需要，审核要求与原生 App 类似。

**Q: 快应用性能如何？**
A: 性能优于小程序，接近原生 App，因为是原生渲染。

**Q: 一个包可以发所有平台吗？**
A: 理论上可以，但各平台有细微差异，建议使用 uni-app 等框架统一适配。

**Q: 快应用如何获取用户？**
A: 主要通过应用商店搜索、负一屏、全局搜索等系统入口。
