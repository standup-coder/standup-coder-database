# 用户增长与 ASO 指南

## 什么是 ASO

```bash
# ASO = App Store Optimization
# 应用商店搜索优化

# 作用：
# - 提升搜索排名
# - 增加自然下载
# - 降低获客成本
```

## ASO vs AS（应用商店优化）

| 项目 | ASO | AS |
|------|-----|-----|
| 全称 | App Store Optimization | App Store |
| 范围 | 搜索 + 推荐 + 榜单 | 搜索为主 |
| 难度 | 中 | 低 |
| 效果 | 长期 | 短期 |

## 搜索排名因素

### 权重排序（参考）

```bash
# 1. 标题关键词
#    - 包含目标关键词
#    - 权重最高

# 2. 关键词字段
#    - 100 字符限制
#    - 逗号分隔

# 3. 下载量和更新频率
#    - 下载量权重高
#    - 近期更新有加成

# 4. 评分和评论
#    - 高评分有帮助
#    - 近期好评重要

# 5. 描述文本
#    - 次要关键词
#    - 影响搜索匹配
```

## 关键词优化

### 标题优化

```bash
# iOS App Store Connect：
# - 主标题：30 字符内
# - 副标题：30 字符内

# 标题公式：
# [品牌名] + [核心关键词] + [亮点/定位]

# 示例：
# 不好：记账本 - 随手记
# 好：随手记 - 专业记账软件 | 3秒记一笔

# 技巧：
# - 核心关键词放前面
# - 用竖线 | 分隔亮点
# - 避免关键词堆砌（会被惩罚）
```

### 关键词字段优化

```bash
# iOS 关键词字段：100 字符

# 策略：
# 1. 研究竞品关键词
# 2. 选择竞争适中、有搜索量的词
# 3. 填写格式：用逗号分隔，不加空格

# 示例：
# 记账,记账本,随手记,账本,预算,理财,存钱,省钱

# 不推荐：
# - 重复关键词（记账,记账,记账）
# - 竞品名称（可能侵权）
# - 无关关键词
```

## 截图与视觉优化

### 截图设计原则

```bash
# 1. 前 3 张最重要
#    - 用户滑动浏览时只看前 3 张
#    - 把核心卖点放前面

# 2. 每张一个卖点
#    - 不要堆砌信息
#    - 一张图说明一个功能

# 3. 使用文字标注
#    - 突出关键数据
#    - 说明功能价值

# 4. 风格统一
#    - 保持配色一致
#    - 使用相同模板
```

### 截图示例结构

```bash
# 第 1 张：品牌 + 核心定位
# - App 名称大字
# - 副标题："3秒记一笔的记账App"

# 第 2 张：核心功能展示
# - 截图 + 功能说明

# 第 3 张：差异化亮点
# - 独特功能或数据

# 第 4 张：用户好评（可选）
# - 用户评价截图

# 第 5 张：使用引导（可选）
# - 注册/下载引导
```

## 评分与评论

### 评论重要性

```bash
# 1. 影响搜索排名
#    - 高评分 App 排名更高

# 2. 影响转化率
#    - 低评分用户不下载

# 3. 影响推荐资格
#    - 精品推荐要求 4.5+ 星

# 目标评分：
# - 4.5+ 星是基本要求
# - 4.7+ 星是优秀
```

### 获取好评技巧

```bash
# 1. 好评触发时机
#    - 用户刚完成正向操作
#    - 如：连续记账 7 天
#    - 如：首次完成目标
#    - 不要在差评后弹出

# 2. 好评弹窗文案
#    - 不要直接要求 5 星
#    - 询问"是否满意"
#    - 满意的再引导好评

# 3. 差评收集
#    - 提供"反馈"入口
#    - 让不满的用户先反馈
#    - 减少公开差评
```

### 代码示例（iOS）

```swift
// 好感评分（SKStoreReviewController）
import StoreKit

@available(iOS 14.0, *)
func requestReview() {
  // 触发时机：用户完成某个正向行为
  if shouldShowReviewPrompt() {
    SKStoreReviewController.requestReview(in: windowScene!)
  }
}

// 自定义好评引导
func showCustomRating() {
  // 检测是否允许弹窗
  if SKStoreReviewController.canRequestReview() {
    // 显示自定义 UI
    showRatingAlert()
  }
}
```

### 差评处理

```bash
# 1. 及时回复
#    - 24 小时内回复
#    - 表示重视

# 2. 专业态度
#    - 不要争辩
#    - 承认问题

# 3. 提供解决方案
#    - 引导联系客服
#    - 提供临时方案

# 回复模板：
# "感谢您的反馈。关于您提到的[问题]，
# 我们正在修复中。如需进一步帮助，
# 请联系 support@example.com"
```

## Apple Search Ads

### 为什么选 Apple Search Ads

```bash
# 1. 用户意图强
#    - 主动搜索
#    - 转化率高

# 2. 精准定向
#    - 关键词定向
#    - 设备/地区定向

# 3. 可见度高
#    - 搜索结果顶部
#    - 带"广告"标识

# 4. 门槛低
#    - $100 最低充值
#    - 按点击付费
```

### Apple Search Ads 入门

#### 第一步：创建账户

```bash
# 1. 访问 https://searchads.apple.com/
# 2. 点击 "Get Started"
# 3. 登录 Apple ID
# 4. 选择账户类型（个人/公司）
# 5. 填写账单信息
```

#### 第二步：创建广告系列

```bash
# 1. 点击 "Campaigns" → "+ New Campaign"
# 2. 选择目标 App
# 3. 设置预算：
#    - 每日预算：$10-$50 起步
#    - 总预算：不限
# 4. 设置出价策略：
#    - Manual（手动）
#    - Automatic（自动）
```

#### 第三步：创建广告组

```bash
# 1. 设置广告组：
#    - 定向关键词（15-25 个）
#    - 定向设备（iPhone/iPad）
#    - 定向地区
#    - 性别/年龄（可选）

# 2. 关键词选择：
#    - 品牌词：App 名称
#    - 品类词：记账、理财
#    - 竞品词：竞品名称

# 3. 出价建议：
#    - 品牌词：$1-2
#    - 品类词：$0.5-1
#    - 竞品词：$0.3-0.5
```

#### 第四步：创建广告创意

```bash
# 1. 上传 App 截图
# 2. 编写标题（最多 30 字符）
# 3. 选择呼吁行动文本：
#    - "下载"
#    - "了解更多"
#    - "立即获取"

# 2. 预览广告效果
```

## 数据分析

### 关键指标

```bash
# 1. 展示次数（Impressions）
#    - 广告/应用被看到的次数

# 2. 点击次数（Clicks）
#    - 用户点击次数

# 3. 点击率（CTR）
#    - Clicks / Impressions
#    - 目标 > 5%

# 4. 下载次数（Downloads）
#    - 实际下载次数

# 5. 下载转化率
#    - Downloads / Clicks
#    - 目标 > 20%

# 6. 成本
#    - CPC（每次点击成本）
#    - CPA（每次获取成本）
```

### 分析工具

```bash
# 1. App Store Connect（iOS）
#    - 免费下载/销量
#    - 收入报告
#    - 活动报告

# 2. Google Play Console（Android）
#    - 安装次数
#    - 评分和评论
#    - 留存数据

# 3. Firebase Analytics（免费）
#    - 用户行为
#    - 事件追踪
#    - 留存分析

# 4. 第三方（付费）
#    - Adjust
#    - AppsFlyer
#    - Branch
```

## 超级个体 ASO 建议

### 优先级排序

```bash
# 高优先级（必须做）：
# 1. 标题优化
# 2. 关键词优化
# 3. 前 3 张截图优化
# 4. 基础评分管理

# 中优先级（有空做）：
# 5. 所有截图优化
# 6. 描述文案优化
# 7. 定期更新维护

# 低优先级（预算充足再做）：
# 8. Apple Search Ads
# 9. 社交媒体推广
# 10. KOL 合作
```

### 常见错误

```bash
# 1. 关键词堆砌
#    - 不要重复写同一个词
#    - 不要写无关热词

# 2. 截图无重点
#    - 不要只是截图
#    - 要有文字说明

# 3. 忽视评分
#    - 不主动收集好评
#    - 不回复差评

# 4. 不更新
#    - 长期不更新影响排名
#    - 定期小版本更新

# 5. 评分操控
#    - 禁止假评论
#    - 禁止诱导好评
#    - 可能导致下架
```
