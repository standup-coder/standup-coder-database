## 应用信息配置

### 创建应用

```bash
# 1. 登录小米开发者平台
#    访问：https://dev.mi.com/
#    使用开发者账号登录

# 2. 选择 "应用分发" → "应用管理"
#    在左侧菜单中找到

# 3. 点击 "上传应用"
#    选择 APK 或 AAB 文件上传
```

### 填写信息

```bash
# 基本信息：
# - 应用名称（必填，≤20字符）
# - 应用分类（必填，一级+二级分类）
# - 应用简介（必填，≤100字符）
# - 详细描述（必填，≤5000字符）
# - 开发者名称

# 截图要求：
# - 截图尺寸：1080x1920 或 720x1280
# - 截图数量：3-8 张
# - 格式：PNG 或 JPG
# - 大小：每张≤10MB
```

### APK/AAB 上传

```bash
# 上传步骤：
# 1. 点击 "上传安装包"
# 2. 选择 APK 或 AAB 文件
# 3. 等待上传完成
# 4. 系统自动检测：
#    - 包名
#    - 版本号
#    - 签名信息
# 5. 填写版本信息
# 6. 提交审核
```

## 审核要点

### 常见拒绝原因

```bash
# 1. 包名问题
- 包名与软著不符
- 包名已被占用
- 包含敏感词汇

# 2. 权限问题
- 权限申请过于频繁
- 权限与功能不符
- 未说明权限用途

# 3. 内容问题
- 应用描述夸大
- 截图与实际不符
- 应用签名不一致
```

### 审核周期

```bash
# 首次审核：1-3 天
# 更新版本：1 天
# 紧急修复：可申请加急
```

## 常见问题

**Q: 小米需要软著吗？**
A: 必须，但接受受理通知书先上线。

**Q: 如何同步到其他厂商？**
A: 在小米平台授权 OPPO、vivo、联想后自动同步。

**Q: 审核被拒怎么办？**
A: 查看拒绝原因，修改后重新提交。

**Q: 应用签名有什么要求？**
A: 使用正式签名，不要使用 debug 签名。

**Q: 小米支持 AAB 吗？**
A: 支持，AAB 和 APK 都可以。
